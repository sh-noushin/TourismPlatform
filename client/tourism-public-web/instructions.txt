.NET backend already exists; build public UI with Next.js (no Exchange UI yet)
Goal

Create a public website with Next.js (App Router) + TypeScript + Tailwind where a user can:

View Houses and Tours

Browse by categories:

HouseTypes

TourCategories

Search/filter like ImmoScout:

Desktop: left filter sidebar + results on right

Mobile: filters open in a drawer

Filters update the URL query string (shareable)

Pagination + sorting

See details pages with photo gallery:

Houses details with photos

Tours details with photos + schedules

Use reusable global colors/styles/components (no repeated CSS)

No login/register in Next.js for now.

Key technical rules

Use App Router (/app directory).

Use design tokens (CSS variables) + Tailwind theme mapping for reusable colors.

Build reusable UI components (components/ui/*) so pages do not repeat CSS.

Prefer Next.js API proxy (/app/api/proxy/...) so browser doesn’t call backend directly (avoids CORS).

Use next/image and configure remotePatterns for backend images.
